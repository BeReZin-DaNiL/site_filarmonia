# Пояснительная записка к курсовому проекту
## Тема: Разработка информационно-справочной системы филармонии

---

## Введение

Целью данной работы является разработка веб-приложения для информационной поддержки деятельности филармонии. Система предназначена для предоставления пользователям актуальной информации о концертах, новостях и услугах филармонии, а также для автоматизации процесса выбора и бронирования билетов.

---

## 1. Постановка задачи

В рамках курсового проекта необходимо разработать программное средство, обеспечивающее выполнение следующих функций:
1.  Просмотр афиши мероприятий с возможностью фильтрации и поиска.
2.  Предоставление детальной информации о каждом концерте (описание, исполнители, программа, стоимость).
3.  Публикация и просмотр новостей филармонии.
4.  Регистрация и авторизация пользователей.
5.  Личный кабинет пользователя (просмотр истории заказов, избранное).
6.  Панель администратора для управления контентом (событиями, пользователями, отчетами).
7.  Адаптивный интерфейс для корректного отображения на мобильных устройствах и десктопах.

---

## 2. Описание предметной области

Предметной областью является деятельность культурно-зрелищного учреждения — филармонии.
Основные сущности предметной области:
*   **Событие (Концерт):** Характеризуется названием, датой, жанром, стоимостью билета, описанием и изображением.
*   **Пользователь (Зритель):** Физическое лицо, использующее систему для получения информации и покупки билетов.
*   **Заказ:** Запись о намерении пользователя посетить определенное мероприятие.
*   **Новость:** Информационное сообщение о деятельности филармонии.
*   **Администратор:** Сотрудник, ответственный за наполнение системы данными.

---

## 3. Актуальность автоматизации

Внедрение информационной системы актуально по следующим причинам:
1.  **Повышение доступности услуг:** Зрители могут узнать репертуар и купить билеты круглосуточно из любой точки мира.
2.  **Снижение нагрузки на кассы:** Часть потока покупателей переходит в онлайн.
3.  **Маркетинговый инструмент:** Возможность оперативно информировать аудиторию о премьерах и акциях через раздел новостей.
4.  **Улучшение имиджа:** Современный и удобный веб-сайт является стандартом для культурных учреждений высокого уровня.

---

## 4. Описание программы

### 4.1. Алгоритмические решения

В основе работы программы лежат следующие ключевые алгоритмы:

1.  **Маршрутизация (Routing):**
    Используется библиотека `react-router-dom`. Алгоритм определяет, какой компонент отобразить в зависимости от URL в адресной строке. Реализована защита маршрутов (`AdminRoute`), которая проверяет роль пользователя перед доступом к административной панели.

2.  **Управление состоянием (State Management):**
    *   Используется `Context API` (`AuthContext`) для хранения глобального состояния авторизации (текущий пользователь, токен).
    *   Локальное состояние компонентов управляется хуком `useState` (например, открытие модальных окон, хранение списка загруженных событий).

3.  **Взаимодействие с сервером (API):**
    Все запросы к серверу инкапсулированы в модуле `api.js`. Используется библиотека `axios`. Реализован механизм перехватчиков (interceptors) для автоматического добавления токена авторизации в заголовки запросов.

4.  **Фильтрация и пагинация:**
    На стороне клиента и сервера реализованы алгоритмы выборки данных (событий) на основе переданных параметров (поисковый запрос, номер страницы).

### 4.2. Описание интерфейса программы

Интерфейс разработан с использованием принципов SPA (Single Page Application) и отзывчивого дизайна (Responsive Web Design).

*   **Главная страница:** Содержит динамический слайдер или сетку карточек с актуальными событиями. Присутствует поиск по названию.
*   **Навигационная панель (Navbar):** Закреплена в верхней части экрана. Содержит ссылки на основные разделы ("Афиша", "Новости", "Контакты") и блок действий пользователя (Вход/Регистрация/Профиль). Адаптируется под мобильные устройства (превращается в "бургер-меню").
*   **Страница события:** Детальное отображение информации с крупным изображением, описанием и кнопкой "Купить билет".
*   **Раздел "Новости":** Список новостных карточек с возможностью перехода к полному тексту статьи.
*   **Личный кабинет:** Интерфейс для просмотра данных пользователя и истории его действий.

### 4.3. Состав приложения

Приложение состоит из двух основных частей (архитектура Клиент-Сервер):

1.  **Frontend (Клиентская часть):**
    *   **Технологии:** React.js, Vite (сборщик), Tailwind CSS (стилизация).
    *   **Назначение:** Отрисовка интерфейса, обработка действий пользователя.

2.  **Backend (Серверная часть - Mock Server):**
    *   **Технологии:** Node.js, Express.
    *   **Назначение:** Эмуляция работы реального API, хранение данных в оперативной памяти (JSON), обработка запросов на авторизацию и получение данных.

## 5. Назначение и состав классов программы

Поскольку приложение написано в функциональном стиле с использованием React Hooks, вместо классических классов здесь используются **Функциональные Компоненты**.

**Основные компоненты (модули):**

1.  **`App.jsx`**: Корневой компонент. Содержит структуру маршрутизации (`Routes`) и определяет общую разметку страницы (Header, Footer).
2.  **`main.jsx`**: Точка входа в приложение. Инициализирует React DOM и провайдеры контекста.
3.  **`Navbar.jsx`**: Компонент навигации. Отвечает за отображение меню и управление состоянием выпадающих списков.
4.  **`Home.jsx`**: Страница афиши. Загружает список событий из API и отображает их.
5.  **`EventDetails.jsx`**: Страница отдельного события. Реализует логику отображения деталей, добавления в избранное и вызова формы бронирования.
6.  **`NewsList.jsx`**: Компонент для отображения списка новостей.
7.  **`NewsDetails.jsx`**: Компонент для отображения полной версии новости.
8.  **`AuthContext.jsx`**: Модуль контекста. Предоставляет методы `login`, `logout` и данные пользователя (`user`) всему дереву компонентов.
9.  **`api.js`**: Сервисный модуль. Содержит функции для HTTP-запросов (`getEvents`, `login`, `getNews` и др.).
10. **`server.js` (Backend)**: Файл сервера. Содержит настройки Express, маршруты API (`/api/events`, `/api/auth`) и логику обработки данных.
